## 🚀 快速开始

1. **新开发者**：查看 `setup/QUICK_START.md`
2. **部署项目**：查看 `setup/` 目录下的部署指南
3. **遇到问题**：查看 `troubleshooting/` 目录
4. **了解修复历史**：查看 `fixes/` 目录

## 📝 文档维护

- 所有修复记录应放在 `fixes/` 目录
- 状态报告应放在 `reports/` 目录
- 新功能实现记录应放在 `implementation/` 目录
- 数据库备份应放在 `database/backups/` 目录
- 日志文件应放在 `logs/` 目录（根目录下）
