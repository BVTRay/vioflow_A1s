# 视频上传功能使用指南

## 快速开始

### 场景1：为现有视频上传新版本 🔄

**适用情况**：客户反馈后需要修改视频，上传迭代版本

**操作步骤**：
1. 在审阅模块找到要更新的视频
2. 点击视频卡片上的 **"上传"** 按钮（📤图标）
3. 系统会自动锁定为"添加新版本"模式
4. 拖拽或点击选择新文件
5. 填写**迭代说明**（必填，如："根据客户反馈调整了配色"）
6. 点击 **"上传为 vX"** 按钮

**特点**：
- ✅ 自动使用原视频的文件名
- ✅ 自动计算下一个版本号
- ✅ 无需手动选择目标视频
- ✅ 保持版本历史完整

---

### 场景2：为项目上传全新视频 🆕

**适用情况**：项目新增视频文件

**操作步骤**：
1. 在审阅模块选择目标项目
2. 点击右上角的 **"上传视频"** 按钮
3. 拖拽或点击选择文件
4. 选择 **"上传新视频"**
5. 填写**新视频说明**（必填，如："项目初稿，等待客户反馈"）
6. 点击 **"上传新视频"** 按钮

**特点**：
- ✅ 自动添加 v1 版本号
- ✅ 文件名自动规范化
- ✅ 可随时查看版本历史

---

### 场景3：为现有视频系列添加新版本（从项目入口） 🔀

**适用情况**：已知项目中有某个视频需要更新，但当前不在该视频卡片位置

**操作步骤**：
1. 在审阅模块选择目标项目
2. 点击右上角的 **"上传视频"** 按钮
3. 拖拽或点击选择文件
4. 选择 **"添加新版本"**
5. 在列表中**选择目标视频**（显示缩略图和当前版本）
6. 填写**迭代说明**（必填）
7. 点击 **"上传为 vX"** 按钮

**特点**：
- ✅ 智能列表：只显示每个视频系列的最新版本
- ✅ 带缩略图，便于识别
- ✅ 自动计算下一个版本号

---

### 场景4：快速上传到近期项目 ⚡

**适用情况**：在工作台快速上传文件到最近的项目

**操作步骤**：
1. 在工作台点击 **"快速上传"** 卡片
2. 从**近期项目列表**中选择目标项目
3. 后续步骤与场景2或场景3相同

**特点**：
- ✅ 显示最近打开的10个活跃项目
- ✅ 显示项目信息（组别、视频数量）
- ✅ 一键快速定位

## 常见问题 FAQ

### Q1: 为什么有些时候不能切换上传类型？

**A**: 当从视频卡片的"上传"按钮进入时，系统会**自动锁定为"添加新版本"模式**，这是为了防止误操作。如果要上传新视频，请从主浏览区右上角的"上传视频"按钮进入。

### Q2: 视频版本号是如何计算的？

**A**: 系统会自动查找同名视频系列的最高版本号，然后在此基础上 +1。例如：
- 如果已有 v1, v2，下一个版本是 v3
- 如果是新视频，版本号自动为 v1

### Q3: "迭代说明"和"新视频说明"有什么区别？

**A**: 
- **迭代说明**：用于描述本次版本相对于上一版本的改动（如："修改了片头动画"）
- **新视频说明**：用于描述新视频的整体内容和用途（如："项目初稿，包含完整剪辑"）

### Q4: 既有视频列表为什么只显示部分视频？

**A**: 为了简化选择，列表只显示每个视频系列的**最新版本**。例如，如果项目中有：
- v1_demo.mp4, v2_demo.mp4, v3_demo.mp4
- v1_intro.mp4, v2_intro.mp4

列表只会显示：
- demo.mp4 (v3) → 上传后将成为 v4
- intro.mp4 (v2) → 上传后将成为 v3

### Q5: 可以上传什么格式的文件？

**A**: 支持：
- 视频文件：MP4, MOV, AVI, MKV 等
- 图片文件：JPG, PNG, GIF 等
- 音频文件：MP3, WAV, AAC 等

### Q6: 上传失败怎么办？

**A**: 请检查：
1. 文件大小是否超过限制（最大 10GB）
2. 网络连接是否正常
3. 是否填写了必填项（说明）
4. 项目状态是否为"活跃"（已定版或已交付的项目无法上传）

如仍有问题，请联系管理员。

### Q7: 如何查看历史版本？

**A**: 在视频卡片上点击版本标识（如 "v3 / 共3版"），即可打开历史版本面板，查看所有版本并切换。

### Q8: 上传后可以修改说明吗？

**A**: 目前上传后的说明不能直接修改。如需记录新的信息，建议：
- 在视频详情中添加批注
- 或在项目备注中记录

## 最佳实践 💡

### 1. 规范的迭代说明

好的示例：
```
✅ 根据客户反馈调整了片头时长（从5s改为3s）
✅ 修复了第10秒处的字幕错误
✅ 完成了包装特效和最终调色
✅ 更新了背景音乐，使用客户指定的版本
```

不好的示例：
```
❌ 修改
❌ v3
❌ 更新了一些内容
```

### 2. 清晰的文件命名

建议使用有意义的文件名：
```
✅ nike_tvc_30s.mp4          → 清晰易懂
✅ product_intro_v1.mp4      → 包含用途信息
✅ apple_launch_final.mp4    → 明确场景
```

避免使用：
```
❌ 未命名.mp4
❌ 新建视频 1.mp4
❌ 123.mp4
```

### 3. 合理使用版本控制

- **小改动**：文字修改、音乐替换 → 添加新版本 ✅
- **大改动**：完全重新剪辑 → 考虑作为新视频 🤔
- **不同版本**：15s版 vs 30s版 → 上传为不同的视频 ⭐

### 4. 定期整理

- 定期归档不需要的旧版本
- 为重要版本添加批注标记
- 保持项目文件整洁有序
